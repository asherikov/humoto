1 Implementing configurable classes {#md_howto_configurable}
2 =================================
3 
4 The framework provides functionality for serialization/deserialization of
5 classes to/from configuration files.
6 
7 
8 In order to implement a configurable class, it is necessary to take two steps:
9 
10 * Inherit from [ConfigurableBase](@ref humoto::config::ConfigurableBase) class,
11  which provides functions to work with configuration files, if they are
12  enabled with corresponding cmake option.
13 
14 * Add '`#include HUMOTO_CONFIG_DEFINE_ACCESSORS`' directive inside a class
15  which includes [define_accessors.h](@ref define_accessors.h) file. This
16  automatically generates functions which are specific for each class or cannot
17  be inherited for technical reasons. Three optional defines can precede this
18  inclusion:
19 
20  - `HUMOTO_CONFIG_SECTION_ID` declares default name of the configuration
21  section corresponding to this class. If it is not defined, the user must
22  implement virtual function which provides this functionality.
23 
24  - `HUMOTO_CONFIG_CONSTRUCTOR` requests generation of additional class
25  constructors for direct initialization from a configuration file.
26 
27  - `HUMOTO_CONFIG_ENTRIES` declares configuration file entries for this
28  class. If not specified the user must implement
29  `readConfigEntriesTemplate()` template function manually.
30 
31 
32 
33 * Implement remaining virtual methods of @ref humoto::config::ConfigurableBase
34  class:
35 
36  - User must provide definition of @ref humoto::config::CommonConfigurableBase::setDefaults()
37  function.
38 
39  - Optionally it is possible to postprocess class members after reading them
40  from a configuration file -- in this case the programmer should provide
41  appropriate implementation of @ref humoto::config::CommonConfigurableBase::finalize()
42  virtual function.
