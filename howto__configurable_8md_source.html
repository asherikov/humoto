1 Implementing configurable classes {#md_howto_configurable}
2 =================================
3 
4 The framework provides functionality for serialization/deserialization of
5 classes to/from configuration files.
6 
7 
8 In order to implement a configurable class, it is necessary to take two steps:
9 
10 * Inherit from [ConfigurableBase](@ref humoto::config::ConfigurableBase) class,
11  which provides functions to work with configuration files, if they are
12  enabled with corresponding cmake option.
13 
14 * Implement pure virtual methods of
15  [ConfigurableBase](@ref humoto::config::ConfigurableBase) class:
16 
17  - Some of these functions can be generated automatically using X-macro
18  technique, if directive '`#include HUMOTO_CONFIG_DEFINE_ACCESSORS`' is added
19  (it includes [define_accessors.h](@ref define_accessors.h) header).
20 
21  - Others must be implemented manually.
22 
23 
24 Optionally it is possible to
25 
26 * Define additional constructors to initialize class from a configuration file
27  using `HUMOTO_DEFINE_CONFIG_CONSTRUCTORS` macro.
28 
29 * Postprocess class members after reading them from a configuration file -- in
30  this case the programmer should provide appropriate implementation of
31  @ref humoto::config::CommonConfigurableBase::finalize() virtual function.
